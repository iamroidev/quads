## Overview
Quads is a full‑stack application consisting of:
- **Mobile** – React‑Native (Expo) client located in `mobile/`
- **Server** – Node.js/TypeScript API located in `server/`
- **Brain** – Utility scripts for data checks and migrations under `brain/`

## Repository Layout
```
quads/
├─ .env                     # environment variables (do NOT commit)
├─ .gitignore               # generated by this repo
├─ README.md                # you are reading it now
├─ mobile/                  # React‑Native app
│   ├─ src/                 # components, screens, services, etc.
│   └─ assets/              # images & icons
├─ server/                  # Express API
│   └─ src/                 # controllers, services, models, routes
├─ brain/                   # ad‑hoc scripts
└─ scripts/                 # deployment & CI helpers
```

## Getting Started
### Prerequisites
- **Node.js** >= 18
- **Yarn** or **npm**
- **Expo CLI** (`npm i -g expo-cli`)
- **Docker** (for server container)

### Mobile
```bash
cd mobile
npm install   # or yarn
expo start
```
### Server
```bash
cd server
npm install   # or yarn
npm run dev   # starts ts-node dev server
```
### Brain Scripts
Each folder under `brain/` contains a `scratch/` directory with TypeScript scripts.
```bash
cd brain/69173bc3-78ac-4a6d-b4bf-4151d0f713ce/scratch
npx ts-node check_orders.ts
```

## CI / Lint / Tests
- **CI** runs on every push (GitHub Actions). It executes linting, unit tests for the server, and builds the mobile app.
- **Lint** uses ESLint + Prettier (see `.eslintrc.json` & `.prettierrc`).
- **Tests**:
  - Server: Jest (`npm run test` in `server/`).
  - Mobile: Jest + React Native Testing Library (`npm run test` in `mobile/`).

## Deployment
- **Mobile** is built and published via Expo EAS (see `.eas/workflows`).
- **Server** is containerised; use `docker build -t quads-server .` inside `server/` and push to your registry.

## Contributing
1. Fork the repo

3. Write tests and lint‑clean code
4. Run `npm run lint:fix` to auto-format
5. Open a PR – CI will run automatically

## Asset Organization
Assets are organized into subfolders under `mobile/assets/`:
- `icons/` – app icons
- `illustrations/` – marketing images
- `splash/` – splash screens

Run `node scripts/organize-assets.js` to auto-sort.
